# Security Policy

## Reporting a Vulnerability

We take the security of CSLearn seriously. If you discover a security vulnerability, please follow these steps:

### How to Report

1. **Do NOT open a public GitHub issue** for security vulnerabilities.
2. Send details to the project maintainers via a **private method** (contact your school administrator or the project maintainer directly).
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (optional)

### What to Expect

- **Acknowledgement**: You will receive a response within 48 hours confirming receipt.
- **Investigation**: The team will investigate and determine the severity and impact.
- **Fix**: A fix will be developed and deployed as soon as possible, depending on severity.
- **Disclosure**: Once a fix is deployed, the vulnerability will be publicly disclosed after a reasonable period.

## Scope

The following areas are in scope for security reporting:

- Authentication and authorization (Supabase Auth integration)
- API routes and data access
- Row Level Security policies
- Environment variable handling
- User data privacy

## Out of Scope

- Third-party services (Supabase, OpenRouter, Vercel) — report directly to them
- Issues requiring physical access to devices
- Social engineering attacks

## Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to fix the issue before public disclosure.
- Make a good faith effort to avoid privacy violations and data destruction.
- Do not exploit the vulnerability beyond what is necessary to demonstrate the issue.

Thank you for helping keep CSLearn and its users safe.
